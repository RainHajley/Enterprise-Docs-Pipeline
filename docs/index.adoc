== Getting Started
Before running commands, make sure you have the Vaultcore CLI installed and targeted at your tenant cluster.

=== Prerequisites
* Vaultcore CLI installed locally.
* Network access to your tenant cluster endpoints.

== Authentication
To connect to the Vaultcore tenant gateway, authenticate with your enterprise service account keys.

1. Run the bootstrap command in your terminal:
+
[source,bash]
----
vaultcore auth login --tenant-id="enterprise-prod-01"
----

2. Verify your session permissions:
+
[source,bash]
----
vaultcore auth status
----

[WARNING]
====
Never store raw API secrets or private tokens in plain-text configuration files or public repositories.
====

== API Integration
The Vaultcore REST API lets you manage security policies, audit logs, and cluster health checks programmatically.

=== Base URL
----
https://api.vaultcore.internal/v1/tenants/{tenant_id}
----

=== Endpoints

[cols="1,2,3", options="header"]
|===
| Method | Endpoint | Description
| `GET` | `/status` | Returns operational health and sync metrics.
| `POST` | `/policies` | Deploys a new security compliance policy.
| `DELETE` | `/cache` | Forces an immediate regional node cache eviction.
|===

=== Example Request

[source,bash]
----
curl -X GET "https://api.vaultcore.internal/v1/tenants/enterprise-prod-01/status" \
     -H "Authorization: Bearer $VAULTCORE_API_TOKEN"
----
