=== Edge Routing and Debug Troubleshooting

If tenant synchronization fails across edge nodes, follow these internal triage steps:

. Inspect the active gateway daemon logs for socket timeout errors:
+
[source,bash]
----
journalctl -u vaultcore-edge -f --since "10 minutes ago"
----

. Flush the regional routing table cache if DNS propagation stalls:
+
[source,bash]
----
vaultcore-cli cache flush --force-region-sync
----

> **Internal Policy:** Always escalate unresolvable node dropouts to the Platform Reliability Team via the internal on-call rotation.
